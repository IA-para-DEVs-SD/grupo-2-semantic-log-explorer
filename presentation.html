O Problema 🔍

  • Logs de produção com milhares de linhas
  • Diagnóstico manual é lento e propenso a erros
  • Stack traces complexos exigem experiência
  • Tempo médio de reparo (MTTR) elevado
  • Informações sensíveis expostas nos logs
⏱️

MTTR Alto

Desenvolvedores gastam horas lendo logs manualmente para encontrar a causa raiz de um erro

2 / 15

A Solução 💡

Uma interface conversacional que permite fazer perguntas em linguagem natural sobre logs, usando IA para identificar erros, causas raiz e sugerir correções.

Chunking Semântico ✂️

Diferente de textos comuns, logs são divididos por eventos e stack traces completos, preservando o contexto do erro intacto.

❌ Chunking Fixo (Tradicional)

Corta o texto a cada N caracteres, quebrando stack traces no meio:

Chunk 1: 2026-03-26 ERROR NullPointer...
Chunk 2: ...Exception at com.app.Service
→ Contexto perdido!

✅ Chunking Semântico (Nosso)

Preserva eventos e stack traces completos como unidades:

Chunk 1: 2026-03-26 ERROR NullPointer
  Exception at com.app.Service.run
  at com.app.Main.main(Main.java:42)
→ Contexto completo!

Segurança e Privacidade 🔒

  • Sanitização de PII: CPFs, e-mails e senhas são mascarados via Regex antes do envio à IA
  • Variáveis de ambiente: Chaves de API gerenciadas via .env, nunca no código
  • Rate Limiting: Middleware que limita 30 requisições por minuto por IP
  • CORS configurado: Apenas origens permitidas podem acessar a API
  • Proteção contra Prompt Injection: Sanitização de inputs do usuário

Exemplo de Sanitização

Antes:
Login falhou para user@email.com
CPF: 123.456.789-00
Senha: s3cr3t_pass

Depois:
Login falhou para [EMAIL]
CPF: [CPF]
Senha: [SENHA]

Prompt Engineering 🤖

O LLM recebe um prompt de sistema especializado como SRE Senior, com instruções de formatação em Markdown.

System Prompt:

Você é um Engenheiro de SRE Senior especializado
em análise de logs de aplicações e infraestrutura.

Formatação:
- Use títulos ### para separar seções
- Use **negrito** para informações críticas
- Coloque trechos de log em blocos de código

Estrutura da Resposta:
### Resumo → ### Erros → ### Causa Raiz → ### Recomendações

Regras:
- Baseie respostas EXCLUSIVAMENTE nos logs fornecidos
- NÃO especule nem invente informações
